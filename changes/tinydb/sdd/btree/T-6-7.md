# T-6.7 — Checkpoint (B6 final task)

## Goal
Add periodic checkpointing. A checkpoint records the set of (LSN, dirty_page_id) pairs that have been flushed to disk, plus an `RT_CKPT` WAL record containing the checkpoint's metadata. On startup, recovery skips frames older than the checkpoint's LSN (T-6.7 is the LAST B6 task; the gate is "100 fuzzy recovery scenarios still pass" — they already do).

## Carry-forward NITs

- v0.1 simplification: no LSN-per-page tracking. Checkpoint just records "the WAL was at LSN N at time of checkpoint; safe to truncate earlier frames on the next successful checkpoint after N".
- Tests still need to pass; all 11 explicit recovery cases + the 100 fuzzy scenarios.
- WAL's `RT_CKPT` code is reserved (`RT_CKPT = 5`) but unused so far.

## Files to Create / Modify

### `src/tinydb/tx/checkpoint.py` (~150 lines)

```python
"""Periodic checkpoint — flush dirty pages + record WAL anchor.

A checkpoint:
1. Iterates over the Pager's dirty pages (buffer-pool tracked).
2. Flushes them via ``pager.write_page`` (which is already idempotent
   because pages are whole).
3. Appends an ``RT_CKPT`` WAL record with the current LSN.
4. (Optional, future) Truncates older WAL frames.

For v0.1 we DO NOT truncate the WAL on checkpoint — that requires
LSN-per-page tracking (which T-6.6 NIT-1 deferred). Instead, the
checkpoint just records a stamp.

API:
    cp = Checkpoint(pager, wal)
    cp.run()    # append RT_CKPT, optionally flush dirty pages
"""
```

### `src/tinydb/tx/__init__.py`
Add `Checkpoint` export.

## Tests Required

`tests/tx/test_checkpoint.py` — at least 4 cases:

1. Empty checkpoint records LSN 0 (or current next_lsn at start).
2. Checkpoint after some DML → LSN advances.
3. Checkpoint is recorded in the WAL (read back via iter_from).
4. Recovery after checkpoint still produces correct state (regression for the 11 explicit cases).
5. Two consecutive checkpoints → WAL has two RT_CKPT records.
6. Checkpoint appends exactly one RT_CKPT (idempotent calls append one each time).
