# T-6.2 — WAL append

## Goal
Implement the append-only Write-Ahead Log file: each frame is a fixed header followed by type byte, payload, and CRC-32C checksum. WAL must support `append(record) -> LSN`, `iter_from(lsn) -> Iterator[WALRecord]`, and detect corruption.

This is the most foundational binary-on-disk structure in B6; everything else builds on it.

## Carry-forward NITs

- Tests use `Pager.open(...)` for fixtures; for T-6.2 the WAL has its **own file** (separate from `.db` and `.idx`), so use `tmp_path / "test.wal"` or similar. No Pager required.
- Zero deps. Use `struct` + `zlib` from stdlib (CRC-32C: Python 3.10+ has `zlib.crc32` (regular) — CRC-32C is in `google.crc32c` package we can't use. So use CRC-32 (`zlib.crc32`) instead; document that deviation in the report).
- File caps: see brief.
- `tinydb.tx.WriteLock`, `WriteLockHeld` already exist from T-6.1.

## File Format (per the task spec)

```
frame:
    [lsn: u64  big-endian]            8 bytes
    [type: u8]                         1 byte
    [payload_len: u32 big-endian]      4 bytes
    [payload: bytes of length payload_len]
    [crc32: u32 little-endian]         4 bytes (over [lsn|type|payload_len|payload])
```

Total header: 13 bytes. Each frame: 13 + payload_len + 4 bytes.

> NOTE: I'm using u32 **big-endian** for payload_len (matches LSN) and u32 **little-endian** for the CRC (matches zlib.crc32 conventions). Document the asymmetry in the report — bikeshed later.

> Pragmatic deviation: original brief used `crc:u32` (no specific polynomial). We use **CRC-32 (ISO 3309, ITU-T V.42)** via `zlib.crc32()` because CRC-32C requires C extension. The detection property (any single-bit flip) is preserved.

"""Write-Ahead Log — append-only log of typed records.

A :class:`WAL` is a thin wrapper over a binary file handle. Records
are appended using :meth:`append`; the file is always grown, never
shrunk (truncation is the checkpoint layer's job, T-6.7).

Frame layout (see module docstring for diagram).
"""
from __future__ import annotations
import struct
import zlib
from dataclasses import dataclass
from pathlib import Path
from typing import BinaryIO, Iterator

# Frame constants
_HEADER_FMT = ">QBI"  # LSN (u64 BE), type (u8), payload_len (u32 BE)
_HEADER_SIZE = struct.calcsize(_HEADER_FMT)
_CRC_FMT = "<I"       # CRC32 little-endian (matches zlib.crc32()'s return)
_CRC_SIZE = struct.calcsize(_CRC_FMT)

#: Record type codes (extensible enum)
RT_BEGIN    = 1
RT_COMMIT   = 2
RT_ROLLBACK = 3
RT_PAGE     = 4   # before-image or after-image page write
RT_CKPT     = 5   # checkpoint anchor

@dataclass(frozen=True, slots=True)
class WALRecord:
    lsn: int           # monotonically increasing; first LSN is 1
    type: int
    payload: bytes

class WAL:
    """Append-only log with CRC32-protected frames.

    Constructs from a path. The file is opened in r+b mode and may
    already have data — call :meth:`recover_lsn` first to skip past
    existing frames on startup. Use ``mode='wb'`` to truncate.
    """

    def __init__(self, path, *, mode: str = "r+b") -> None:
        self._path = Path(path)
        self._fp = self._path.open(mode)
        self._next_lsn = self._recover_lsn() if mode == "r+b" else 1

    def append(self, type: int, payload: bytes) -> int:
        """Append a frame. Returns the LSN assigned."""
        ...

    def iter_from(self, lsn: int) -> Iterator[WALRecord]:
        """Yield records starting at LSN >= lsn. Stop on corruption."""
        ...

    def truncate_to(self, lsn: int) -> None:
        """Truncate the file so records with LSN >= lsn are removed.
        Used by checkpoint (T-6.7).
        """
        ...

    def fsync(self) -> None: ...

    def close(self) -> None: ...

    @property
    def next_lsn(self) -> int: ...

    @property
    def size(self) -> int: ...

## Tests Required

`tests/tx/test_wal.py` — at least 8 cases:

1. Empty WAL → `next_lsn == 1`.
2. Append one record → returns LSN == 1; `next_lsn == 2`.
3. Append three records → LSNs are 1, 2, 3; payload bytes round-trip.
4. `iter_from(0)` yields all records.
5. `iter_from(2)` yields records 2 and 3 only.
6. CRC corruption: write a valid frame, then seek and flip one bit in the payload → `iter_from` raises `WALCorruptionError` (new exception class).
7. Truncated last frame at the tail (e.g. write 5 frames, then truncate the last to half its size) → iter_from stops cleanly without raising.
8. `truncate_to(2)` removes records ≥ 2; `next_lsn == 2` and `iter_from(0)` returns just record 1.
9. `close()` then `WAL(path)` reopens and recovers LSN correctly.
10. (bonus) `fsync()` is callable and doesn't raise.
11. (bonus) Empty payload → still adds a 17-byte frame (header + CRC).
12. (bonus) Type byte 200 (unknown type) is preserved on read; client decides what to do.
