# T-6.4 — Constraint enforcement on commit

## Goal
Wire the executor's DML paths so that UNIQUE / NOT NULL / type violations during a transaction cause the WHOLE transaction to roll back (per REQ-TRX-3) — not just raise. Also add an **integration test** that runs a real DML sequence inside `mgr.transaction()` and verifies constraints trigger rollback.

## Carry-forward NITs

- `TransactionManager` (T-6.3) exposes `mgr.transaction() -> ctx` and handles auto-COMMIT/ROLLBACK via context manager.
- DML plans (`Insert`/`Update`/`Delete`) already raise `ConstraintViolation` via `IndexManager.on_insert()` and `NotNullViolation` via `_assert_not_null()` (B5 contract). For T-6.4 the txn manager just needs to catch these and route them to rollback.
- Coverage gate 85% on `tinydb/tx/`.

## Files to Modify / Create

### `src/tinydb/tx/manager.py` (extend)

Add a `run_dml(action: Callable[[TransactionContext], T]) -> T` helper that:
1. Acquires the write lock (implicit via `transaction()`).
2. Runs the callable; if it raises `ConstraintViolation` / `NotNullViolation` / `TypeMismatchError`, the manager rolls back (auto via exception path) and re-raises as `tinydb.errors.ConstraintViolation` (or the subclass).
3. Otherwise commits.

Or: just **integrate** with the existing `transaction()` context — the auto-rollback-on-exception already handles the cases correctly. So T-6.4 is mostly about the integration test, not new code in the manager.

### `tests/tx/test_constraints.py` (NEW)

```python
"""Integration tests — DML inside TransactionManager enforces constraints."""
import threading

def _setup_db(tmp_path):
    pager = Pager.open(tmp_path / "test.db")
    catalog = Catalog(pager)
    catalog.create_table("users", [
        Column("id", TypeTag.Int, primary_key=True, unique=True, not_null=True),
        Column("name", TypeTag.Text, not_null=True),
        Column("age", TypeTag.Int),
    ])
    wal_path = tmp_path / "test.wal"
    wal = WAL(wal_path, mode="wb")
    wal.close()
    wal = WAL(wal_path)
    mgr = TransactionManager(pager, wal)
    return pager, catalog, mgr, wal

def test_unique_violation_rolls_back_transaction(tmp_path):
    pager, catalog, mgr, wal = _setup_db(tmp_path)
    indexer = IndexManager(catalog, pager)
    executor = Executor(catalog, pager, indexer=indexer)
    plan_insert = lambda: executor.execute(plan(parse("INSERT INTO users VALUES (1, 'alice', 30)"), catalog))

    # First insert succeeds
    with mgr.transaction():
        plan_insert()
    # Second insert with duplicate id raises AND rolls back the second
    with pytest.raises(ConstraintViolation):
        with mgr.transaction():
            plan_insert()
    # Table still has 1 row
    rows = executor.execute(plan(parse("SELECT * FROM users"), catalog))
    assert len(rows) == 1
```
