# T-5.4 — Sort + Limit

## Goal
Make ORDER BY sort rows and LIMIT/OFFSET slice them. Pure in-memory operation (heap-fetch all → sort → slice).

## Carry-forward NITs to incorporate (T-5.1 + T-5.2 + T-5.3)

1. `Executor` lives at `tinydb.executor.executor`.
2. `Plan.table` is a `@property` returning the base table for any plan. **Sort and Limit must inherit from src.table.**
3. Tests use `Pager.open(...)` + `Catalog(pager)` (no Engine).
4. `Sort` already has `limit` + `offset` fields; `Limit = Sort` (alias). Brief says narrow `Limit` into a dedicated plan; do so by **adding a new `Limit` dataclass** and keeping `Limit = Sort` re-export working. Recommended: introduce `Limit(src, limit, offset)` and have the planner emit that. The existing `Sort` keeps `limit=None, offset=0` for "sort with optional slice" legacy support — but in T-5.4 the planner always separates Sort (sort-only, no limit/offset) and Limit (slice-only). The existing tests are the place to flip this.
5. Coverage gate still 85% on `src/tinydb/executor/`.
6. `Limit` IS NOT a `Sort` alias after T-5.4 — it must be its own dataclass but still importable as `tinydb.executor.Limit`.

## Files to Modify

### `src/tinydb/executor/ops.py` — narrow Sort / Limit

```python
@dataclass(frozen=True, slots=True, kw_only=True)
class Sort(Plan):
    """In-memory sort by ``keys``. No LIMIT here (T-5.4 split)."""
    src: Plan
    keys: Sequence[tuple]   # (column_name, descending: bool)
    op_name: str = "Sort"

    @property
    def table(self) -> str:
        return self.src.table

    def open(self, ctx: Executor) -> Iterator[tuple]:
        rows = list(self.src.open(ctx))
        # resolve name → index via the same name_to_idx the Filter op uses
        col_idx = ctx.name_to_idx(self.table)
        for col, desc in self.keys:
            if col not in col_idx:
                raise UnknownColumnError(f"{self.table}.{col}")
        rows.sort(key=lambda r: tuple(
            (r[col_idx[c]] is None, r[col_idx[c]]) if not desc
            else (r[col_idx[c]] is None, _neg(r[col_idx[c]]))
            for c, desc in self.keys
        ))
        yield from rows


@dataclass(frozen=True, slots=True, kw_only=True)
class Limit(Plan):
    """Slice the upstream plan to ``limit`` rows after skipping ``offset``."""
    src: Plan
    limit: int
    offset: int = 0
    op_name: str = "Limit"

    @property
    def table(self) -> str:
        return self.src.table

    def open(self, ctx: Executor) -> Iterator[tuple]:
        if self.limit < 0:
            raise ValueError("LIMIT must be non-negative")
        if self.offset < 0:
            raise ValueError("OFFSET must be non-negative")
        rows = list(self.src.open(ctx))
        yield from rows[self.offset : self.offset + self.limit]
```

Edge cases:
- Multi-key sort: tie-breaker follows declaration order.
- NULL ordering: v0.1 simple rule — NULL sorts **last** in ASC, **first** in DESC (consistent with SQLite default; not configurable). Document this in the report's deviations.
- `_neg` for DESC: define a tiny helper in the same file or in a utility section; for int/float/str we negate comparables. Unknown types: define a wrapper that handles `None` (already filtered to last/first) and otherwise raises `TypeError` if subtraction fails — surface to user.
- Empty `keys` (rare; some legacy paths) — treat as identity (no sort).

### `src/tinydb/executor/planner.py` — split Sort from Limit

The planner currently emits one Sort-with-limit-offset for both ORDER BY and LIMIT. Split:
- If `stmt.order_by` is non-empty → wrap in `Sort(src, [(o.column, o.descending) for o in stmt.order_by])`.
- If `stmt.limit` is non-None → wrap in `Limit(src, limit=stmt.limit.limit, offset=stmt.limit.offset)` AFTER any Sort.
- Order: Project → Sort → Limit → outer.

Drop the `Limit = Sort` alias at the bottom of `ops.py` (replace with the real dataclass).

### `src/tinydb/executor/executor.py`
The executor needs `name_to_idx(table_name) -> dict[str, int]` already (Filter uses it). Confirm it works for Sort too; otherwise extend minimally. **Do not** modify `Executor.execute`'s public surface.

### `src/tinydb/executor/__init__.py`
Re-export `Sort`, `Limit` (already exported). No new names.

## Tests Required

1. ASC by single int column (`ORDER BY age`) → rows in ascending age.
