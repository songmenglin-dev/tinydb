# T-7.2 — End-to-end SQL flows

## Goal
Write `tests/integration/test_e2e.py` with 20+ real SQL scenarios that exercise **multiple capabilities together** (DDL + DML + Index + Transaction + Recovery + Aggregate). This is the integration gate for Batch 7.

## Carry-forward NITs

- `tinydb.open(path)` + `db.execute(sql)` (T-7.1 API) is the entry point.
- All lower-level modules (PAGER, BUFFERPOOL, CATALOG, INDEX, WAL, TX, EXECUTOR) are wired and tested independently.
- Tests use `tmp_path` to avoid file collisions.

### `tests/integration/test_e2e.py` (~400 lines)

20+ scenarios. Examples (illustrative, not prescriptive):

1. Create table → INSERT 100 rows → SELECT all → row count matches.
2. CREATE TABLE with PRIMARY KEY + NOT NULL; reject INSERT violating PK.
3. CREATE TABLE with UNIQUE; reject INSERT duplicating the unique key.
4. CREATE INDEX on a column → SELECT WHERE col = literal uses IndexScan (assert via EXPLAIN-like inspection if exposed; otherwise assert correctness on a big table).
5. CREATE UNIQUE INDEX; two INSERTs duplicate → second rolls back.
6. CREATE TABLE → INSERT 10 rows → UPDATE age = age + 1 → SELECT.
7. CREATE TABLE → INSERT 100 rows → DELETE WHERE age > 50 → SELECT count.
8. CREATE TABLE → INSERT 10 rows → ORDER BY name DESC LIMIT 5.
9. CREATE TABLE → INSERT 5 rows → SELECT MIN / MAX / SUM / AVG / COUNT.
10. CREATE TABLE → INSERT 10 rows (5 in dept 'A', 5 in 'B') → GROUP BY dept → COUNT(*) per group.
11. CREATE TABLE → INSERT 10 rows → BEGIN / INSERT / COMMIT / close + reopen → SELECT shows the new row.
12. CREATE TABLE → INSERT / BEGIN / INSERT / ROLLBACK / close + reopen → SELECT shows only first INSERT.
13. CREATE TABLE → INSERT / close (no COMMIT) / reopen with recovery → recovery sees autocommit and the row is there.
14. CREATE TABLE → INSERT 1 row → DROP TABLE → SELECT raises (UnknownTableError).
15. CREATE TABLE 5 columns → INSERT in column-list form (subset).
16. CREATE TABLE → SELECT WHERE col IS NULL → matches NULLs.
17. CREATE TABLE → SELECT WHERE col LIKE 'abc%' → matches prefix.
18. CREATE TABLE → INSERT 1000 rows → close + reopen → SELECT all → row count matches.
19. CREATE TABLE → INSERT 10 → UPDATE 5 → DELETE 2 → SELECT shows correct residual.
20. CREATE TABLE with DECIMAL column → INSERT DECIMAL value → roundtrip.
21. CREATE TABLE with DATE column → INSERT DATE '2024-01-15' → roundtrip.
22. CREATE TABLE → multiple BEGIN/COMMIT cycles → row counts consistent.

Plus 1-2 cross-table flows:
23. CREATE TABLE a, CREATE TABLE b → INSERT both → SELECT joined? **NO** — JOIN is out of scope per DP-0; skip.

24. CREATE TABLE → INSERT then UNIQUE conflict → tx rolls back → SELECT shows empty (the rest of the in-flight group also reverted).

These are 22 scenarios. Drop / adjust as needed to reach ≥20.
