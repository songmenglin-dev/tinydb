# T-7.1 — Database class (the public API)

## Goal
Build the `tinydb.Database` class that wraps the full set of internals (`Pager` + `BufferPool` + `Catalog` + `IndexManager` + `WAL` + `TransactionManager` + `Executor`) into a single, ergonomic public API.

This is the layer every user interacts with — B8 (CLI) and external Python callers will both use it.

## Carry-forward NITs

- `bind_heap` still reads `catalog._pager` (private). The Engine will expose `.pager` publicly (T-7.1 finally lands this).
- Autocommit tx_id=0 sentinel must be documented.
- WAL truncation deferred (carry to B9.
- Public surface per B5: `Executor`, `plan`, `Plan`, `SeqScan`, `IndexScan`, `Filter`, `Project`, `Sort`, `Limit`, `Insert`, `Update`, `Delete` (already exportable from `tinydb.executor`).
- Errors already exportable from `tinydb.errors`: `TinydbError`, `ParseError`, `ConstraintViolation`, `NotNullViolation`, `TypeMismatchError`, `WALCorruptionError`.

### `src/tinydb/api.py` (~250 lines — main file)

```python
"""High-level Database facade.

This is the only public entry point a user of tinydb needs to know.
Hides the lower-level modules (Pager, BufferPool, Catalog, IndexManager,
WAL, TransactionManager, Executor) behind a small, ergonomic surface.

Usage:
    with tinydb.Database("/tmp/test.db") as db:
        db.execute("CREATE TABLE users (id INT PRIMARY KEY, name TEXT NOT NULL)")
        db.execute("INSERT INTO users VALUES (1, 'alice')")
        rows = db.execute("SELECT * FROM users WHERE id = 1")
        # rows == [(1, 'alice')]
"""
from __future__ import annotations
from contextlib import contextmanager
from pathlib import Path
from typing import Any, Sequence, Union
from tinydb.storage.pager import Pager
from tinydb.storage.buffer_pool import BufferPool
from tinydb.storage.catalog import Catalog
from tinydb.types.system import Column
from tinydb.sql.parser import parse
from tinydb.executor.executor import Executor
from tinydb.executor.planner import plan as _plan
from tinydb.index.manager import IndexManager
from tinydb.tx.wal import WAL
from tinydb.tx.manager import TransactionManager
from tinydb.tx.recovery import Recovery
from tinydb.errors import TinydbError, ParseError

Result = Union[list[tuple], list[tuple[int, ...]]]

class Database:
    """The user-facing database handle.

    Construct with a path; the database file is opened (or created).
    Call :meth:`execute` for DDL/DML; :meth:`transaction` for an
    explicit BEGIN/COMMIT/ROLLBACK block; :meth:`close` to release
    the file handle.  Use as a context manager for automatic close.
    """

    def __init__(self, path: Union[str, Path]) -> None:
        self._path = Path(path)
        self._pager = Pager.open(self._path)
        self._buffer = BufferPool(self._pager)
        self._catalog = Catalog(self._pager)
        # Open or create the WAL alongside the database.
        wal_path = self._path.with_suffix(self._path.suffix + ".wal")
        if wal_path.exists():
            self._wal = WAL(wal_path)
        else:
            # Touch an empty WAL so subsequent opens work.
            wal_path.write_bytes(b"")
            self._wal = WAL(wal_path)
        # Run crash recovery before any in-process use.
        Recovery(self._wal, self._pager).replay()
        self._indexer = IndexManager(self._catalog, self._pager)
        self._txn = TransactionManager(self._pager, self._wal)
        self._executor = Executor(
            catalog=self._catalog,
            pager=self._pager,
            indexer=self._indexer,
            mgr=self._txn,
        )

    @property
    def pager(self) -> Pager:
        """Public handle to the Pager (T-7.1 surfaces the previously-
        private ``catalog._pager`` accessor as documented in T-5.2 NIT-10
        and T-5.5 NIT-3)."""
        return self._pager

    @property
    def catalog(self) -> Catalog:
        return self._catalog

    @property
    def executor(self) -> Executor:
        return self._executor

    def execute(self, sql: str) -> Result:
        """Run a single SQL statement.

        SELECT returns rows as ``list[tuple]``.  DML returns a single
        ``[(affected_count,)]`` row.  DDL returns ``[]``.
        """
        stmt = parse(sql)
        return self._executor.execute(_plan(stmt, self._catalog, indexer=self._indexer))

    @contextmanager
    def transaction(self):
        """Run a BEGIN/COMMIT/ROLLBACK block.

        Auto-COMMIT on clean exit; auto-ROLLBACK on exception.
        """
        with self._txn.transaction() as tx:
            yield tx

    def close(self) -> None:
        """Close the WAL and Pager.  Idempotent."""
        if getattr(self, "_closed", False):
            return
        self._wal.close()
        self._pager.close()
        self._closed = True

    # Context manager sugar
    def __enter__(self) -> "Database":
        return self

    def __exit__(self, et, ev, tb) -> bool:
        self.close()
        return False


# Module-level singleton-style helper for convenience.
def open(path: Union[str, Path]) -> Database:
    """Open or create a database.  Equivalent to ``Database(path)``."""
    return Database(path)
```
