# T-5.5 — INSERT / UPDATE / DELETE execution + index maintenance

## Goal
Make the three DML plans actually write data — to Heap, with IndexManager hooks firing so indexes stay in sync, and the executor enforcing schema constraints (NOT NULL, UNIQUE).

This is the batch that closes the read/write loop end-to-end.

## Carry-forward NITs to incorporate (T-5.1..T-5.4)

1. `Executor` lives at `tinydb.executor.executor` and carries `catalog` + optionally `IndexManager`.
2. `Plan.table` is `@property` returning base table. DML plans already have `.table` as a field (they are leaves).
3. `Heap` is bound by reading `catalog._pager` and rebinding `Heap._head_pid = meta.heap_pid` (T-5.2 NIT). T-5.5 may want a tiny helper `Heap.attach(pager, head_pid)` — feel free to introduce one in this batch (file caps allow it).
4. `coerce_value(value, tag) -> (bytes, TypeTag)` already exists (`src/tinydb/types/coerce.py`).
5. `encode_row` / `decode_row` already exist (`src/tinydb/types/codec.py`).
6. `IndexManager.on_insert(table, rid, row)`, `.on_update(table, rid, old, new)`, `.on_delete(table, rid, row)` already enforce UNIQUE and skip NULLs (B4 contract).
7. `Heap.delete(rid)` writes a tombstone (B2 contract) so `scan()` skips it.

### `src/tinydb/executor/heap_bind.py` (NEW, optional)
```python
from tinydb.storage.pager import Pager
from tinydb.storage.heap import Heap

def bind_heap(catalog, table_name: str) -> Heap:
    meta = catalog.get_table(table_name)
    pager = getattr(catalog, "_pager", None)
    if pager is None:
        raise RuntimeError(
            "executor needs a Pager; pass engine.pager or catalog._pager"
        )
    heap = Heap(pager)
    heap._head_pid = meta.heap_pid  # rebind to the catalog-owned chain
    return heap
```

This extracts the T-5.2 NIT-2 / NIT-10 logic into one helper. Tests for INSERT/UPDATE/DELETE use this.

> ```python
> def encode_row_coerced(values: Sequence[Any], tags: Sequence[TypeTag]) -> bytes:
>     """Encode a row with per-column coercion (used by INSERT)."""
>     out = bytearray()
>     for v, t in zip(values, tags):
>         blob, actual_tag = coerce_value(v, t)

>     return bytes(out)
> ```

`Update.open()`:
```python
def open(self, ctx: Executor) -> Iterator[tuple]:
    heap = bind_heap(ctx.catalog, self.table)
    meta = ctx.catalog.get_table(self.table)
    tags = tuple(c.tag for c in meta.columns)
    name_idx = {c.name: i for i, c in enumerate(meta.columns)}
    affected = 0
    for rid in list(heap.scan()):  # snapshot — update may relocate
        blob = heap.read(rid)
        if blob is None:
            continue
        old_row = decode_row(blob, tags)
        if self.predicate is None or eval_expr(self.predicate, old_row, name_idx, tags):
            new_row = list(old_row)
            for col, val in self.assignments:
                if col not in name_idx:
                    raise UnknownColumnError(f"{self.table}.{col}")
                col_def = meta.columns[name_idx[col]]
                if col_def.not_null and val is None:
                    raise NotNullViolation(...)
                new_row[name_idx[col]] = val
            new_blob = encode_row_coerced(new_row, tags)
            # overwrite in place (Heap has no "update" — delete + insert with same Rid?)
            # Simplest v0.1: delete old, insert new, get new Rid.
            heap.delete(rid)
            new_rid = heap.insert(new_blob)
            if ctx.indexer is not None:
                # The OLD row's rid is `rid`; the NEW row's rid is `new_rid`.
                # This means index entries under `rid` for the OLD key were
                # already removed by delete(). Update must remove old
                # (under old row/r) THEN insert new (under new row/new_r).
                ctx.indexer.on_delete(self.table, rid, old_row)
                ctx.indexer.on_insert(self.table, new_rid, tuple(new_row))
            affected += 1
    yield (affected,)
```

`Delete.open()`:
```python
def open(self, ctx: Executor) -> Iterator[tuple]:
    heap = bind_heap(ctx.catalog, self.table)
    meta = ctx.catalog.get_table(self.table)
    tags = tuple(c.tag for c in meta.columns)
    name_idx = {c.name: i for i, c in enumerate(meta.columns)}
    affected = 0
    for rid in list(heap.scan()):
        blob = heap.read(rid)
        if blob is None:
            continue
        row = decode_row(blob, tags)
        if self.predicate is None or eval_expr(self.predicate, row, name_idx, tags):
            heap.delete(rid)
            if ctx.indexer is not None:
                ctx.indexer.on_delete(self.table, rid, row)
            affected += 1
    yield (affected,)
```

> Document in the report: v0.1 does an in-place overwrite via delete + insert (Rid changes). T-5.5 NIT: this may break FK-like referential integrity if it ever lands; for B5 scope it's fine. T-7 (Public API) may want a real in-place update.

## Tests Required

`tests/executor/test_dml.py` — at least 12 cases:

1. `INSERT INTO users VALUES (1, 'alice', 30)` then `SELECT` returns the row.
2. `INSERT` with NULL in nullable column → row stored, NULL comes back.
3. `INSERT` with NULL in NOT NULL column → raises `NotNullViolation`.
4. `INSERT` into non-existent table → raises `UnknownTableError` (already in planner).
5. `UPDATE users SET name = 'bob' WHERE id = 1` → row updated; `SELECT` returns new value; old value gone.
6. `UPDATE users SET age = NULL WHERE id = 1` with NOT NULL → `NotNullViolation`.
7. `UPDATE users SET name = X WHERE NOT (id = 1)` → only the matching row updates.
8. `UPDATE users SET X = Y` (no predicate) → all rows affected.
9. `DELETE FROM users WHERE id = 1` → row gone.
10. `DELETE FROM users` (no predicate) → all rows gone, table empty.
11. UNIQUE constraint: `CREATE UNIQUE INDEX` then `INSERT` duplicate → `ConstraintViolation`.
12. Index maintenance end-to-end: build index on `users(name)`, INSERT 3 rows, UPDATE one name, DELETE one row → `SELECT WHERE name = ...` reflects the index state (no orphans, no missing).
13. Row count return: `INSERT INTO t VALUES (...)` yields `(1,)`; `DELETE FROM t WHERE ...` yields `(N,)` where N = affected.
