### `src/tinydb/executor/ops.py`
```python
from __future__ import annotations
from dataclasses import dataclass, field
from typing import Any, Callable, Iterator, Optional, Sequence

# A row is a tuple of bytes-encoded column values in declared column order.
Row = tuple

@dataclass(frozen=True, slots=True)
class Plan:
    """Marker base — concrete plans are subclasses below."""
    op_name: str

    def open(self, ctx: "Executor") -> Iterator[Row]: ...
    def __iter__(self) -> Iterator[Row]:
        return self.open(None)  # type: ignore[arg-type]

@dataclass(frozen=True, slots=True)
class SeqScan(Plan):
    table: str

@dataclass(frozen=True, slots=True)
class IndexScan(Plan):
    table: str
    index: str
    lo: Any = None
    hi: Any = None
    lo_inclusive: bool = True
    hi_inclusive: bool = True

@dataclass(frozen=True, slots=True)
class Filter(Plan):
    src: Plan
    predicate: object  # Expr from tinydb.sql.ast

@dataclass(frozen=True, slots=True)
class Project(Plan):
    src: Plan
    columns: Sequence[str]

@dataclass(frozen=True, slots=True)
class Sort(Plan):
    src: Plan
    keys: Sequence[tuple[str, bool]]  # (column, desc)
    limit: Optional[int] = None
    offset: int = 0

# aliases to satisfy the brief
Limit = Sort  # T-5.4 will narrow

@dataclass(frozen=True, slots=True)
class Insert(Plan):
    table: str
    values: tuple  # row tuple to insert

@dataclass(frozen=True, slots=True)
class Update(Plan):
    table: str
    assignments: Sequence[tuple[str, object]]
    predicate: object  # Expr | None

@dataclass(frozen=True, slots=True)
class Delete(Plan):
    table: str
    predicate: object
```

## Planner Behavior (T-5.1 scope — the brief must implement these exactly)

1. `Select`:
   - Lower bound: `SELECT * FROM t` → `SeqScan(table=t.name)` only.
   - WHERE clause (top-level BinaryOp / unary) → wrap in `Filter(seq, expr)`.
   - Projection (`SELECT a, b FROM t`) → wrap in `Project(seq_or_filter, ['a','b'])`.
     * `SELECT *` produces a `Project` listing every column of `t` (in declared order) — keep parity for executor dispatch.
   - ORDER BY → wrap in `Sort(filter_or_project, [('c', False)])` (False=ASC).
     * Multi-key supported (T-5.4 will execute; planner already produces the right plan).
   - LIMIT / OFFSET → set `Sort.limit` / `Sort.offset`. If no ORDER BY, still produce a Sort node with empty `keys` (T-5.4 treats empty keys as identity order).
   - GROUP BY (only the structural plan; aggregate execution is T-5.6): when the SELECT list contains an `Aggregate`, wrap the source in a marker (T-5.1: keep `Plan` shape generic — just preserve seq/filter; raise NotImplementedError at execute time).
   - For T-5.1: GROUP BY / aggregate *execution* may raise NotImplementedError on execute; the **plan tree itself must still be constructible** (no error from `plan()`).
2. `Insert(values=...)` → `InsertPlan(table=t.name, values=...)`.
3. `Update(assignments, predicate)`:
   - No predicate → wrap nothing (executor scans all rows); T-5.1: produce `UpdatePlan(table=t.name, assignments=..., predicate=None)`.
   - Predicate → keep as-is on the plan.
4. `Delete(predicate)` → `DeletePlan(table=t.name, predicate=...)`.

5. **Validation (gates the executor would otherwise hit later)**:
   - Unknown table → `UnknownTableError(table_name)`.
   - Unknown column (referenced in WHERE / projection / assignment / OrderBy key) → `UnknownColumnError(table.col)`.

6. **Index-scan selection** lives in T-5.3. For T-5.1 the planner should:
   - Detect when WHERE column has a usable single-column index (EQ or range), call into a stub `_try_index_plan(...)` that returns None for now (T-5.3 fills in). Code path MUST exist with the right shape — `plan_select` checks `_try_index_plan(filter_predicate, table_meta)` first and uses the returned `IndexScan` if non-None.
