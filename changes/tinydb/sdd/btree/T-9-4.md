# T-9.4 — Scope audit

## Goal
Verify that NO out-of-scope features have leaked into tinydb. The Out-of-Scope list (from `changes/tinydb/proposal.md > Out`):
- JOIN queries
- Concurrency (multi-thread / multi-process safety beyond single-writer)
- ALTER TABLE
- Views
- Triggers
- Foreign keys
- Network / client-server mode

## How

Run grep / pattern checks against `src/` and `tests/`. Document each finding. If a "violation" surfaces, document the deviation in the report (it's NOT a blocker — could be a benign word like "trigger" appearing in a help text).

### Concrete checks (loose — flag, don't fail):

```bash
# 1. JOIN across tables (in executor parser / planner):
grep -rE "\bJOIN\b|\bjoin\b" src/tinydb/executor/

# 2. ALTER TABLE in sql/parser.py:
grep -rE "ALTER" src/tinydb/sql/

# 3. CREATE VIEW:
grep -rE "CREATE VIEW|create_view|CreateView" src/tinydb/

# 4. Trigger:
grep -rE "TRIGGER|trigger" src/tinydb/

# 5. Foreign key:
grep -rE "FOREIGN KEY|REFERENCES" src/tinydb/

# 6. Network (sockets, http):
grep -rE "import socket|import http|import urllib|import requests" src/tinydb/

# 7. Threading beyond WriteLock (T-6.1):
grep -rE "threading\." src/tinydb/ | grep -v "tx/lock.py"
```

## File to Create

### `tests/scope_audit.md` (~50 lines)

A human-written audit log: each check + result (✓ none found / ⚠ benign occurrence explained).
