## Goal

Implement `BTreeIndex.delete(key, rid) -> None` with full rebalance:

1. **Locate and remove** the `(key, rid)` entry from its leaf.
2. **Underflow detection** — if the leaf drops below the minimum occupancy, rebalance.
3. **Sibling borrow** — if a sibling has entries to spare, move one across.
4. **Merge** — if neither sibling can spare, merge the leaf with a sibling.
5. **Recursive upward rebalance** — if a merge causes the parent to underflow, repeat at the parent.
6. **Root collapse** — if the root is an internal node with a single child, replace the root with that child.

## B-tree order, minimum occupancy, and merge threshold

The brief uses **order N=64** per design D-4 (`tinydb/design.md`). For B-tree of order N:
- Maximum entries per leaf: `2N - 1 = 127` (current `_write_leaf` raises `BTreeOverflowError` past this; the split path uses `mid = len(keys) // 2` which is well-defined regardless of N)
- Maximum children per internal: `2N = 128`
- **Minimum entries per leaf: `N - 1 = 63`**
- **Minimum children per internal: `N = 64`**

Underflow = below the minimum. The rebalance algorithm:
1. **Borrow**: if a sibling has `> min` entries, move one across the separator key.
2. **Merge**: otherwise, concatenate the underflowing node with the sibling, dropping the separator key from the parent.
3. **Root collapse**: if the parent is the root and now has 1 child, replace the root.

The minimum-occupancy threshold is exposed as a module-level constant `_MIN_LEAF_ENTRIES: int = 63` (or whatever the order is) in `btree.py`. Tests assert against the constant.

## Design decisions the implementer must make

1. **No-op vs KeyError on missing (key, rid).** The brief is silent. Recommendation: **silent no-op** (matches Python `dict.pop(key, None)` semantics for the "key exists but value not present" case). Document the choice in the docstring.
2. **Borrow direction.** When a leaf underflows, prefer the **left sibling** for borrowing (smaller separator stays stable). If no left sibling, try right. Document.
3. **Merge direction.** When both siblings are at the minimum, prefer to merge with the **right sibling** (consistent direction). Document.
4. **Constant exposure.** Define `BTreeIndex.ORDER = 64` (class attribute) and `BTreeIndex.MIN_LEAF_ENTRIES = ORDER - 1` so tests can import and assert against them. Use these constants in the actual underflow check.
5. **Recursive helper signature.** `_delete_from_subtree(pid, node, key, rid) -> tuple[bool, bool]` returns `(removed, underflow_at_parent)`. Caller acts accordingly.
