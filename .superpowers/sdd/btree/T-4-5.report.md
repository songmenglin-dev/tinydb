# T-4.5 Report — Composite (tuple) key indexes

## Status: DONE

## What I implemented

T-4.5 extends `BTreeIndex` to accept **tuple keys** for composite (multi-column) indexes. Python's natural tuple comparison is lexicographic, so the existing `_lower_bound` / `_upper_bound` helpers Just Work on tuples without changes. The task was a mix of tests + minimal docs + a small targeted code fix for the prefix-bound case.

### Code changes (minimal)

**`src/tinydb/index/btree_internal.py`** — new helper:
- `_is_past_upper(key, hi, inclusive, *, prefix_mode=False)`: standard `key > hi` (or `key >= hi`) comparison with `TypeError` handling. When `prefix_mode=True` and `hi` is a strict tuple prefix of `key`, returns `False` (the key is in the prefix range). This enables `range(("Smith",), ("Smith",), inclusive=True)` to yield every entry whose key starts with `("Smith",)`.
- Added to `__all__`.

**`src/tinydb/index/btree.py`** — small modifications to the public API:
- `search()`: wrapped in `try/except TypeError → return []` so a scalar key against a tree of tuple keys returns empty rather than raising. Added tuple-key note to docstring.
- `range()`: detects `lo == hi` and enables `prefix_mode` in `_is_past_upper` for prefix-bound semantics. Wrapped in `try/except TypeError` so scalar range bounds against a tree of tuple keys yield nothing. Added tuple-key + prefix-bound note to docstring.
- `insert()` / `delete()`: docstring note that `key` may be a scalar or tuple.
- Class docstring: removed "composite keys (T-4.5)" from the deferred-tasks list.

### Test additions

**`tests/index/test_btree.py`** — 8 new composite-key tests:

1. `test_composite_range_with_full_tuple_keys` — `range(("Smith", "A"), ("Smith", "Z"))` returns the two Smith entries.
2. `test_composite_prefix_range_returns_matching_prefix` — `range(("Smith",), ("Smith",), inclusive=True)` returns all 3 Smith entries; `inclusive=False` returns `[]`.
3. `test_composite_cross_prefix_range_lex_order` — `range(("Smith",), ("Tay",))` returns Smiths only (Tay excluded, standard half-open).
4. `test_composite_search_by_full_tuple_hit` — `search(("Smith", "Alice"))` returns the matching rid.
5. `test_composite_search_by_full_tuple_miss` — `search(("Smith", "Zelda"))` returns `[]`.
6. `test_composite_search_scalar_against_tuple_keys_returns_empty` — `search(1)` against `[(1, "a"), (1, "b")]` returns `[]` (not TypeError).
7. `test_composite_range_scalar_bounds_against_tuple_keys_returns_empty` — `range(0, 5)` against tuple keys returns `[]` (not TypeError).
8. `test_composite_delete_by_full_tuple_removes_entry` — `delete(("Smith", "Alice"), rid)` removes the right entry; prefix range after delete returns the surviving Smith.
