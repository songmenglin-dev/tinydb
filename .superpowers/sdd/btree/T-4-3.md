## Tree-walk algorithm

### `search(key)` ŌĆö O(log n)

```python
def search(self, key) -> list[Rid]:
    self._ensure_loaded()
    pid = self._root_pid
    node_view = self._root_view  # already loaded
    while isinstance(node_view, InternalNode):
        child_idx = _find_child(key, node_view)
        pid = node_view.children[child_idx]
        node_view = self._read_node_view(pid)   # type byte + bytes
    # node_view is now a LeafNode (or raise if on-disk corruption)
    if not isinstance(node_view, LeafNode):
        raise RuntimeError(f"expected leaf at pid={pid}, got {type(node_view).__name__}")
    leaf = node_view
    start = self._lower_bound_key(leaf.keys, key)
    end = self._upper_bound_key(leaf.keys, key)
    return list(leaf.rids[start:end])
```

### `range(lo, hi, *, inclusive=True)` ŌĆö O(log n + k)

```python
def range(self, lo, hi, *, inclusive=True) -> Iterator[Rid]:
    if lo > hi:
        return
    self._ensure_loaded()
    # Descend to the leftmost leaf whose key is >= lo.
    pid = self._root_pid
    node_view = self._root_view
    while isinstance(node_view, InternalNode):
        child_idx = _lower_bound_keys(node_view.keys, lo)  # first separator >= lo
        # child may contain entries < lo; descend anyway, range will trim.
        pid = node_view.children[child_idx]
        node_view = self._read_node_view(pid)
    # Walk forward leaf-by-leaf.
    leaf_pid = pid
    while leaf_pid != NO_NEXT:
        leaf = _read_leaf_from_bytes_or_pid(...)  # or self._leaf_cache[leaf_pid]
        for i in range(len(leaf.keys)):
            k = leaf.keys[i]
            if k < lo:
                continue
            if inclusive:
                if k > hi:
                    return
            else:
                if k >= hi:
                    return
            yield leaf.rids[i]
        leaf_pid = leaf.next_leaf_pid
```

Important: the existing T-4.2 `range()` already walks the sibling chain. T-4.3's upgrade is that the **starting leaf is found via tree descent**, not by blindly reading the root. For single-leaf indexes the behaviour is identical; for multi-leaf indexes the tree walk ensures correctness when `lo` doesn't lie in the leftmost leaf.
