## What was implemented

### Public API
- `BTreeIndex.delete(key, rid) -> None` — replaces the `NotImplementedError`
  placeholder.  Behaviour:
  - Silent no-op when `(key, rid)` is absent (matches
    `dict.pop(key, None)` semantics for an absent value under an
    existing key).
  - On leaf underflow, prefers borrowing from the left sibling
    (keeps the parent separator stable); falls back to merging with
    the right sibling (right wins; the child page becomes orphan).
  - Recurses upward: a merge drops a separator key from the parent,
    so the parent itself may now underflow.  Borrow/merge repeat at
    each level.
  - Root collapse: if the root is an internal node left with a
    single child after a merge, that child takes over as root.
  - Duplicate keys: only the specific `rid` under `key` is removed;
    other rids for the same key remain in insertion order.

## Design decisions implemented

1. **Missing `(key, rid)` is a silent no-op** — matches
   `dict.pop(key, None)` semantics.  Documented in `delete()`.
2. **Borrow direction: prefer left sibling** — keeps the parent
   separator moving downward in the key order.
3. **Merge direction: prefer right sibling, fall back to left** —
   right page wins; the child page becomes orphan.
4. **Constants exposed** as module globals (`btree.ORDER`,
   `btree.MIN_LEAF_ENTRIES`, etc.) AND as class attributes on
   `BTreeIndex` (`BTreeIndex.ORDER`, etc.) so tests and callers can
   reference either side.  Tests assert against the class-attribute
   form.
5. **Recursive helper signature**:
   `delete_from_subtree(pid, node, key, rid) -> tuple[bool, bool]`
   returns `(removed, underflow_at_self)`.  Caller acts accordingly.
