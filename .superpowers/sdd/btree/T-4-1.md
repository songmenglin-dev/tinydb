## Page layout for a leaf node

Page (4 KB = `pager.page_size`):

```
offset 0: u32  next_leaf_pid   (sibling chain; 0xFFFFFFFF = end)
offset 4: u16  key_count       (number of (key, rid) pairs)
offset 6: u16  reserved        (zero)
offset 8: leaf payload starts
```

Payload format: a sequence of fixed-size entries, each entry = `[u16 key_len][key_bytes][u32 page_id][u16 slot_id]`.

- key_len ≤ 65535 (key encoding per `TypeTag` via `tinydb.types.codec.encode_value` / `decode_value`)
- Order target: 64 entries per leaf page → fits comfortably under 4 KB even with text keys up to 64 chars.

Use the existing `tinydb.types.codec` module for key (de)serialization — do NOT hand-roll a codec.
