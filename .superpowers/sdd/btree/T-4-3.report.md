## What was implemented

### Tree-walking API
- `BTreeIndex.search(key) -> list[Rid]` — descends root → internal → leaf via `_find_child`, scans matching entries. Replaces the T-4.1 placeholder `NotImplementedError`.
- `BTreeIndex.range(lo, hi, *, inclusive=True) -> Iterator[Rid]` — descends to the leftmost leaf whose key is `>= lo` (via `_lower_bound` on internal separator keys), then walks the sibling chain via `next_leaf_pid` until the key crosses `hi`.

### NIT-cleanup (all 5 from T-4.2 review)
1. ✅ Removed unused `key_type` parameter from `_read_leaf` and `_read_internal`. Added `_read_leaf_from_bytes` / `_read_internal_from_bytes` helpers that accept already-read bytes.
2. ✅ Promoted all internal `assert` statements to `RuntimeError`. `python -O -m pytest tests/` green confirms migration is complete (no assert was silently stripped under `-O`).
3. ✅ Removed `# type: ignore[assignment]` in `_descend_to_first_leaf`. Replaced with explicit `RuntimeError` on `None`.
4. ✅ Avoided double-page-read by introducing `_read_leaf_from_bytes` / `_read_internal_from_bytes` helpers. `_read_node_view` reads the page once, dispatches on `page[0]`, and passes the bytes through.
5. ✅ `_split_internal` happy path covered by `test_split_internal_happy_path_produces_valid_children` (500 long-Text inserts force both leaf and internal-node splits).

### Refactor for the line cap
Extracted the two static bisect helpers (`_lower_bound`, `_upper_bound`) from `BTreeIndex` to module-level functions in `btree_internal.py` (they're general ordered-key helpers). `btree.py` now 385 lines (was 407, was 376 in T-4.2).

### Split helpers extracted
The `_split_leaf` and `_split_internal` static methods remain on `BTreeIndex` for now — they're short (50 lines combined) and tied to the BTreeIndex instance via `pager` and `key_type`. Splitting them into a separate module is T-4.4+ territory (where delete + rebalance live).

## Recovery note

The first T-4.3 dispatch was terminated by a 429 API rate-limit error mid-run ("token plan 上限"). The implementer agent reported "All 35 tests passing" before the API error cut it off. On manual verification, 5 of 35 tests were actually failing — all due to **stale test imports** (`from tinydb.index.btree import _read_leaf` after `_read_leaf` was correctly removed from `btree.py` and moved to `btree_leaf.py`).

Recovery (5-line fix):
- `tests/index/test_btree.py:291` — split import to `from tinydb.index.btree import BTreeIndex, NO_NEXT` + `from tinydb.index.btree_leaf import _read_leaf`
- `tests/index/test_btree.py:401` — split to `from tinydb.index.btree import BTreeIndex` + `from tinydb.index.btree_internal import _read_internal`
- `tests/index/test_btree.py:642` — split to `from tinydb.index.btree import BTreeIndex, InternalNode` + `from tinydb.index.btree_internal import _read_internal`
- `tests/index/test_btree.py:712` — changed `from tinydb.index.btree import _read_leaf` to `from tinydb.index.btree_leaf import _read_leaf`
- `tests/index/test_btree.py:732` — changed `from tinydb.index.btree import _read_internal` to `from tinydb.index.btree_internal import _read_internal`

Then 3 callsites in `src/tinydb/index/btree.py` had stale `self._lower_bound` / `self._upper_bound` after those staticmethods were removed; updated to the imported module-level functions.

All 35 tests now pass; 433/433 full suite (both modes); 95% coverage on `src/tinydb/index/`.
